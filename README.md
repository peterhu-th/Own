# 记忆窝点 (Own)

一个私密的双人日记本微信小程序，致力于提供一个没有外界干扰、只属于两个人的静谧记录空间。

## 功能特性

- **专属私密空间**：通过 6 位专属邀请码彼此绑定，绑定成功后，日记内容仅两人可见。
- **时光日记本**：
  - **发布与回溯**：支持“记录此刻”和“捡拾过去”。
  - **富媒体记录**：支持长文本、最多 9 张图片，以及语音日记。
  - **便捷管理**：支持修改和删除日记。
- **互动批注**：可以给自己和对方的日记批注。
- **实时情感连接**
  - **在线状态感知**：实时显示对方是否在线以及累积陪伴时长。
  - **时光轴动态**：以 Timeline 形式记录双方的互动。

## 技术栈

- **前端框架**：[uni-app](https://uniapp.dcloud.net.cn/) + **Vue 3** (Composition API) + **Vite**
- **状态管理**：**Pinia**
- **后端支撑**：**微信云开发 (WeChat Cloud Base)**
  - 云函数 (Cloud Functions)：承担所有核心业务逻辑及权限校验。
  - 云数据库 (Cloud Database)：NoSQL 灵活存储。
  - 云存储 (Cloud Storage)：存储用户上传的图片及头像。

## 项目架构与云函数划分

整个项目的服务端逻辑由高度解耦的云函数构成，确保了权限的严格隔离：

1.  **`authCloud` (鉴权中心)**
    - 负责静默登录、初始化用户数据。
    - 负责生成 24 小时有效期的邀请码、处理双方绑定逻辑。
    - 处理全局的心跳轮询，计算并更新双方的在线时长 (`total_online_minutes`) 和最后活跃时间。
2.  **`diaryCloud` (日记中枢)**
    - 负责日记的发布、修改、删除及点赞模块。
    - 负责各个维度日记列表（个人、两人、树洞、喜欢的日记）的获取。
    - 全面采用复合游标分页，提示性能。
    - 严格的读取与查询权限校验，从服务端级别防止越权访问私密日记。
3.  **`commentCloud` (互动模块)**
    - 处理针对特定日记的批注增加与删除。
    - 内置基于目标日记可见性的严格越权拦截。
4.  **`userCloud` (用户与动态模块)**
    - 获取个人配置、拉取伴侣实时状态。
    - 更新用户头像（`avatarUrl`）与昵称（`nickname`）。
    - 获取双方的时间轴历史动态 (`ActivityLog`)。

## 本地运行与部署

### 1. 环境准备

- 安装 Node.js (推荐 v16+)。
- 下载并安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)。
- 在微信公众平台注册一个小程序，获取 `AppID`。

### 2. 项目配置

1.  修改项目根目录下 `project.config.json` 中的 `appid` 为您自己的 AppID。
2.  进入 `src/App.vue`，将 `uni.cloud.init` 里的 `env` 替换为您自己的云开发环境 ID。

### 3. 安装依赖与运行

```bash
npm install
npm run dev:mp-weixin
```

编译完成后，在微信开发者工具中导入项目根目录下的 `dist/dev/mp-weixin` 目录即可预览。

### 4. 云端部署

在微信开发者工具的“云开发”面板中：

1.  创建所需的数据集合（Collections）：`Users`, `Diaries`, `Comments`, `Likes`, `ActivityLog`。
2.  **设置数据权限**：为了前端能顺利通过云函数读取图片和基础配置，建议将各集合权限设置为“所有用户可读，仅创建者可读写”（因为核心的读写逻辑都在云函数中跨越安全规则执行）。
3.  依次右键 `cloudfunctions` 目录下的云函数文件夹，选择 **“上传并部署：云端安装依赖(不上传node_modules)”**。

## 性能与架构优化说明

- **复合游标分页**：彻底抛弃了传统的 `.skip()`，全面采用包含 `create_time/timestamp` 与 `_id` 的双重条件复合游标。不仅解决了随着数据量增长造成的深度翻页慢查询问题，更规避了极端并发下同毫秒时间戳造成的“数据漏登”隐患。
- **并发健壮性**：日记详情页的多重异步拉取（日记正文、评论列表、点赞状态）加入了外层 `try-catch` 容错及内部超时竞速 (`Promise.race`)。即使遇到服务端局部节点异常或超时，也能优雅降级显示主内容，防止前端白屏卡死。
- **前端异步竞态管控**：重构了页面生命周期的异步加载顺序。强制令用户鉴权及基本信息的获取先于核心业务数据的拉取完成，从根本上杜绝了因接口返回时差导致的 UI 状态及权限判断错位。
- **按需加载与 UI 收纳**：针对公开列表全面使用 `.field()` 过滤掉非必要的长文本字段，大幅缩减请求体积。对个人主页的时间轴采用默认折叠的现代 UI 排版，提升高频选项的首屏触达率。
