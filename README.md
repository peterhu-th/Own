# 记忆窝点 (Own)

一个私密的双人日记本微信小程序，致力于提供一个没有外界干扰、只属于两个人的静谧记录空间。

## 功能特性

- **专属私密空间**
  - 通过 6 位专属邀请码进行双人绑定，绑定成功后，私密日记内容仅你们两人可见。
  - 支持单方面申请解绑，内置 72 小时冷静期机制，解绑前支持一键“打包我们的回忆”。
- **时光日记本**
  - **多样化记录**：支持长文本撰写、最多 9 张图片上传，以及一键点击极速录音。
  - **排版与视效**：自适应长文本智能折叠与九宫格图片自动收纳。
- **匿名广场**
  - 可将日记设为公开，以匿名身份发布至广场，与陌生人分享生活。
  - **沉浸式阅读**：广场采用卷首语底部固定与底部渐隐遮罩的沉浸式滚动设计。
- **互动与连接**
  - **批注与点赞**：支持给自己、对方或广场日记进行批注留言与实时点赞互动。
  - **在线感知**：在个人主页提供清晰的专属伴侣状态栏，实时展示伴侣当前是否在线与累积陪伴时长。
  - **时光轴**：以 Timeline 形式记录双方的互动轨迹。
- **数据导出**
  - 广场及多处列表均支持多选模式，一键将选中的日记无损导出为 TXT 文本，方便永久珍藏。

## 技术栈

- **前端框架**：[uni-app](https://uniapp.dcloud.net.cn/) + **Vue 3** (Composition API) + **Vite**
- **状态管理**：**Pinia**
- **后端支撑**：**微信云开发 (WeChat Cloud Base)**
  - 云函数 (Cloud Functions)：承担所有核心业务逻辑及权限校验。
  - 云数据库 (Cloud Database)：NoSQL 灵活存储。
  - 云存储 (Cloud Storage)：存储用户上传的图片及头像。

## 架构与核心设计

整个项目的服务端逻辑由高度解耦的云函数构成，确保了权限的严格隔离：

1. **`authCloud` (鉴权中心)**
   - 负责静默登录、初始化用户数据。
   - 负责生成有效期邀请码、处理双方绑定与解绑逻辑。
   - 处理全局心跳轮询，计算并更新双方的在线时长 (`total_online_minutes`) 和最后活跃时间。
2. **`diaryCloud` (日记中枢)**
   - 负责日记的发布、修改、删除、点赞及批量导出。
   - 负责各个维度日记列表（个人、两人、广场、喜欢的日记）的获取。
   - 全面采用 **复合游标分页**，解决深度翻页慢查询，规避并发导致的数据漏登。
   - 严格的读取与查询权限校验，从服务端级别防止越权访问私密日记。
3. **`commentCloud` (互动模块)**
   - 处理针对特定日记的批注增加与删除。
   - 内置基于目标日记可见性的严格越权拦截。
4. **`userCloud` (用户与动态模块)**
   - 获取个人配置、拉取伴侣实时状态。
   - 更新用户头像（`avatarUrl`）与昵称（`nickname`）。
   - 获取双方的时间轴历史动态 (`ActivityLog`)。

## 本地运行与部署

### 1. 环境准备

- 安装 Node.js (推荐 v16+)。
- 下载并安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)。
- 在微信公众平台注册一个小程序，获取 `AppID`。

### 2. 项目配置

1. 修改项目根目录下 `project.config.json` 中的 `appid` 为您自己的 AppID。
2. 进入 `src/App.vue`，将 `uni.cloud.init` 里的 `env` 替换为您自己的云开发环境 ID。

### 3. 安装依赖与运行

```bash
npm install
npm run dev:mp-weixin
```

编译完成后，在微信开发者工具中导入项目根目录下的 `dist/dev/mp-weixin` 目录即可预览。

### 4. 云端部署

在微信开发者工具的“云开发”面板中：

1. 创建所需的数据集合：`Users`, `Diaries`, `Comments`, `Likes`, `ActivityLog`。
2. **设置数据权限**：建议将各集合权限设置为“所有用户可读，仅创建者可读写”，因为核心读写逻辑均在云函数中越权执行。
3. 依次右键 `cloudfunctions` 目录下的云函数文件夹，选择 **“上传并部署：云端安装依赖(不上传node_modules)”**。
